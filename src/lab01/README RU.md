# О проекте
## Класс Product
В этом проекте реализован пользовательский класс **Product**, моделирующий товар в системе онлайн-продаж. Класс содержит собственный атрибут: "store_name", определяющий название магазина, к которому относятся товары; а также атрибуты объектов: название товара, цена, наличие на складе и модель товара. Также реализовано логическое состояние товара (активный или неактивный).
### Атрибут класса:

* store_name: атрибут для присвоения имени магазину.

### Атрибуты экземпляра (приватные поля):
* _name - название продукта
* _price - цена продукта
* _estoque - количество на складе
* _model - модель продукта
* _is_active - статус продукта (активен/деактивирован)
### Свойства (@property):
* Чтение: name - название продукта
* Чтение и запись: price - цена продукта
* Чтение: estoque - количество на складе
* Чтение: model - модель продукта
* Чтение: is_active - статус продукта (активен/деактивирован)
### Магические методы:
* `__str__` — Удобное представление информации о продукте, понятное пользователю
* `__repr__` — техническое представление продукта — только для разработчиков
* `__eq__` — сравнение продуктов по названию и модели
### Бизнес-методы:
* deactivate() - деактивирует продукт
* eactivate() - активирует продукт
* apply_discount(percentage) - применяет скидку к товару
* sell(self, quantity) - продает товар

# Демонстрация проекта

**2 . Аудит инвентаря (Сердце логики)**
* **Инвентаризация с вложенным циклом for:** Проходя по списку inventory и сравнивая каждый элемент со следующими, программа создает реальный сценарий контроля качества, проверяя, какие товары имеют одинаковое имя и модель..
* **Взаимодействие с __eq__:** команда "if inventory[i] == inventory[j]", заставляет Python автоматически выполнять код, написанный в *[model.py](model.py)*, вызывая предупреждение (Warning) и предложение объединить остатки (stock). Это автоматизирует управление магазином.

![Realização de Negócios](../../images/lab01/Realização%20de%20Negócios.png)

**5. Манипуляция атрибутами класса и экземпляра**
* **Атрибуты класса:** Изменение атрибута класса через сам класс влияет на все его экземпляры. Это считается «глобальным изменением».
* **Атрибуты экземпляра:** Изменение атрибута класса через конкретный экземпляр (объект) приводит к тому, что изменения отражаются только в этом объекте, не затрагивая остальные. Это считается «локальным изменением». В проекте было протестировано изменение названия магазина с использованием атрибута класса и атрибута экземпляра через общий шаблон, то есть через класс (Product.store_name) или через конкретный объект (p3.store_name).
* **Окончательный статус наличия товара:** проект работает стабильно и позволяет обновлять информацию о наличии товара в магазине и текущем статусе продукции.
